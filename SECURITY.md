# 安全策略 / Security Policy

## 📌 支持版本

| 版本 | 支持状态 |
|------|----------|
| 1.0.x | ✅ 安全更新支持 |

## 🐛 报告安全漏洞

发现安全漏洞时，请通过以下方式报告：

### 方式一：GitHub Security Advisory（推荐）

1. 访问仓库的 **Security** 标签页
2. 点击 **Report a vulnerability**
3. 填写漏洞详情

### 方式二：发送邮件

发送邮件至：security@enterprise-risk.com

### 报告内容

请提供以下信息：

- 漏洞类型（SQL 注入、XSS、CSRF 等）
- 受影响版本
- 复现步骤
- 潜在影响
- 修复建议（可选）

## 🔒 安全响应流程

1. **接收报告** - 24 小时内确认收到
2. **评估漏洞** - 3 个工作日内评估严重程度
3. **修复漏洞** - 根据严重程度制定修复计划
4. **发布更新** - 修复后发布新版本
5. **公开披露** - 修复后 30 天公开漏洞详情

## 🛡️ 安全最佳实践

### 开发者的责任

- 使用参数化查询防止 SQL 注入
- 对所有用户输入进行验证和转义
- 使用 HTTPS 传输敏感数据
- 定期更新依赖包
- 不在代码中硬编码密钥

### 用户的责任

- 定期更新到最新版本
- 修改默认密码
- 配置防火墙规则
- 定期备份数据
- 监控异常访问

## 📅 安全更新时间表

- 严重漏洞：24 小时内发布热修复
- 高危漏洞：7 天内发布修复
- 中危漏洞：30 天内发布修复
- 低危漏洞：下个版本修复
